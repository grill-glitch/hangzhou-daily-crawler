## 修复 XOR 章节渲染错误

**问题：**
- 文件：`src/exclusive-or.rst`
- 错误：在 `:math:` 角色内错误使用 `\verb*|...|` 命令
- 警告信息："Can't find closing delimiter for \verb"（多次出现）

**原因：**
`:math:` 是 Sphinx 的数学角色，用于渲染 LaTeX 数学公式。`\verb` 是 LaTeX 的逐字命令，两者不兼容，导致解析器无法识别闭合定界符。

**修复：**
```diff
- 例如（:math:`\verb*| |` 符号表示空格）：
- -  :math:`\verb*| the |` 及其变体如 :math:`\verb*|. The |`
- -  :math:`\verb*| of |` 及其变体
- -  :math:`\verb*| to |` 及其变体
- -  :math:`\verb*| and |` （无变体；只出现在句子中间）
- -  :math:`\verb*| a |` 及其变体
+ 例如（空格用 `` `` 符号表示）：
+ -  `` the `` 及其变体如 ``. The ``
+ -  `` of `` 及其变体
+ -  `` to `` 及其变体
+ -  `` and `` （无变体；只出现在句子中间）
+ -  `` a `` 及其变体
```

**修复后：**
- 重新编译，警告从 134 个降至 60 个
- 所有 `\verb` 相关错误消失
- 已重新部署到生产环境

## 修复数学命令 \madd 和 \lll 渲染

**问题：**
- 文件：`src/stream-ciphers.rst`
- 未定义命令：`\madd`（模加运算）、`\lll`（左旋转）
- 现象：MathJax 无法渲染这些符号

**根因：**
`conf.py` 中的 `mathjax3_config` 只定义了 `\xor` 宏，未定义 `\madd` 和 `\lll`。

**首次修复尝试：**
```python
'madd': '{\\boxplus}',   # ❌ 外层花括号导致递归错误
'lll': '{\\lll}',        # ❌
```
- 编译时出现 "MathJax maximum macro substitution count exceeded" 错误

**递归错误原因：**
`\lll` 本身是 LaTeX 的原始命令（`<<<` 符号）。定义 `'lll': '\\lll'` 会导致：
1. `\lll` 展开为 `\\lll`
2. `\\lll` 中的 `\lll` 再次触发宏展开
3. 无限递归，达到 MathJax 的最大替换次数限制

**最终修复：**
将 `\lll` 映射到 `\\ll`（`<<` 符号），避免递归同时保留左移语义：
```python
mathjax3_config = {
    'tex': {
        'macros': {
            'xor': '\\oplus',
            'madd': '\\boxplus',  # 模加（modular addition）
            'lll': '\\ll',        # 左旋转（使用 << 符号）
        }
    }
}
```

**结果：**
- ✅ 编译成功（警告 105 个）
- ✅ 无递归错误
- ✅ `\madd` 正确渲染为 `⊞`
- ✅ `\lll` 正确渲染为 `≪`
- ✅ HTML MathJax 配置自动包含新宏
- ✅ 已重新部署到 `https://crypto.notarobot.ggff.net`
- ✅ 在线验证：无 `merror` 或 "maximum macro substitution" 错误
